# 提案：git-domain-e2e-handoff-hardening

## 做什么

- 为 Git 领域建立端到端回归套件与交接标准模板。
- 固化“任意时刻可交接”的文档流程与检查清单。

## 为什么做

没有 E2E 收口与交接模板，阶段性改动容易在跨人接力时失真，导致最终目标无法稳定落地。

## 怎么做

- 增加 E2E：subagent isolated_write -> worktree/copy -> patch handoff -> auto-apply -> archive 回收。
- 每个变更文档必须包含：做什么、为什么做、怎么做、验收标准、相关文档。
- 增加离开前检查：当前阶段、下一步、阻塞点、回归命令。

## 非目标

- 不在本阶段扩展新的业务功能。
- 不在本阶段重构现有协议版本。

## 验收标准

- E2E 套件可重复通过。
- 新接手者仅依赖文档可独立继续推进。
- 交接检查清单执行后无“下一步不明确”问题。
