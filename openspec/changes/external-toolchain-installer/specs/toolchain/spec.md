# 规格增量：toolchain（external installer integration）

## 新增要求

### 要求：Toolchain 安装能力必须支持通用外部安装器

`omne toolchain bootstrap` 必须支持调用“调用方无关”的独立安装器 CLI。

#### 场景：外部安装器可用

- 给定环境中可找到外部安装器可执行文件
- 当执行 `omne toolchain bootstrap --json`
- 则调用方通过外部安装器完成 git/gh 准备
- 且输出仍包含兼容字段 `schema_version`、`target_triple`、`managed_dir`、`items`

#### 场景：外部安装器不可用

- 给定外部安装器不可用
- 当执行 `omne toolchain bootstrap --json`
- 则返回可诊断的失败信息或受控回退行为
- 且不得静默成功

### 要求：外部安装器必须保持调用方无关

外部安装器输出与行为不得绑定 `omne-agent` 专有上下文。

#### 场景：跨调用方复用

- 给定任意符合契约的调用方
- 当调用安装器 CLI
- 则可仅依据公开 JSON 字段消费结果
- 且不依赖 `omne-agent` 私有路径、事件或进程协议

### 要求：app-server 不得承载安装实现

`crates/app-server` 不得引入 git/gh 安装实现细节。

#### 场景：边界扫描

- 给定源码仓库
- 当执行安装实现关键字扫描
- 则 `crates/app-server` 中不包含安装下载/解压/验签核心逻辑

### 要求：外部网关必须防开放代理滥用

用于安装下载的网关实现必须是固定路由白名单，不允许任意代理。

#### 场景：非法路由

- 给定包含任意 URL 参数的请求（例如 `?url=`）
- 当请求命中网关
- 则响应拒绝（`403` 或等价拒绝语义）
- 且不会转发到任意第三方地址

#### 场景：合法白名单路由

- 给定合法工具/版本/平台组合的固定路由
- 当请求命中网关
- 则返回重定向或等价的受控来源响应
- 且目标域名属于白名单集合

## 变更要求

### 要求：安装来源为公共可验证资源

外部安装器必须仅使用公共可访问且可验证的资源来源。

#### 场景：来源审计

- 给定安装流程日志或结构化输出
- 当审计下载来源
- 则来源属于官方发布地址或声明的公共镜像
- 且不依赖私有文件托管服务
