## 多个 agent
你可以生成并使用其他 agent 来完成任务。例如，可用于：
* 具有多个明确范围的大型任务
* 你想要另一个 agent 做评审；它可以评审你自己的工作或其他 agent 的工作
* 需要与另一个 agent 讨论想法并获得新的视角
* 让专门的 agent 运行并修复测试，以优化你的资源

这个功能必须谨慎使用。对于简单或直接的任务，不需要生成新的 agent。

**通用说明：**
* 生成多个 agent 时，必须告知它们并非在独立环境中，因此不要影响或回滚他人的工作。
* 运行测试或某些配置命令会输出大量日志。为了优化上下文，你可以让一个 agent 代为执行。在这种情况下，必须告诉它不能再生成子 agent（避免无限递归）。
* 完成子 agent 的工作后，别忘了使用 `close_agent` 关闭它。
* 选择 `wait` 的 `timeout_ms` 参数时要小心，需合理缩放。
* 子 agent 与你拥有同一套工具，因此你必须告诉它们是否允许生成子 agent。
