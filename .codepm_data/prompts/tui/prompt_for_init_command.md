生成一个名为 AGENTS.md 的文件，作为该仓库的贡献者指南。
你的目标是产出清晰、简洁、结构良好的文档，使用描述性标题，并为每个章节提供可执行的说明。
遵循下面的大纲，但可按需调整——相关的就添加，不适用的可省略。

文档要求

- 文档标题为 "Repository Guidelines"。
- 使用 Markdown 标题（#, ## 等）组织结构。
- 保持简洁，建议 200-400 词。
- 说明简短、直接，并且针对本仓库。
- 需要时给出示例（命令、目录路径、命名模式）。
- 语气保持专业、指导性。

建议章节

项目结构与模块组织

- 概述项目结构，包括源代码、测试和资源所在位置。

构建、测试与开发命令

- 列出构建、测试和本地运行的关键命令（如 npm test、make build）。
- 简要说明每个命令的作用。

代码风格与命名规范

- 指定缩进规则、语言风格偏好和命名模式。
- 包含使用的格式化或 lint 工具。

测试指南

- 指出测试框架和覆盖率要求。
- 说明测试命名规范及运行方式。

提交与 Pull Request 指南

- 总结项目 Git 历史中的提交信息规范。
- 概述 PR 要求（描述、关联 issue、截图等）。

（可选）如有需要，可增加其他章节，例如安全与配置提示、架构概览或 Agent 专用说明。
