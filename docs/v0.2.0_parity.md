# v0.2.0：Codex 功能对齐 + 跨项目精华 TODO

> 目标：**先把 Codex 的“可被编排的 Agent CLI”完整做出来**（协议/事件/执行/沙箱/审批/工具/skills/MCP），再在此之上叠加 CodePM 的并发编排与交付适配（Git/远程等都只是 adapter）。
>
> 硬约束：**git/workspace 不以 Docker 为前提**（我们用 `/tmp`/worktree 等目录隔离；不禁止 agent 自己运行 Docker/Podman）。
>
> 原则：先把数据结构与边界做对；“功能对齐”不是复制粘贴，而是把设计抽象成更干净、更可测、更可维护的 Rust 系统，并把其它项目的优点落到接口与 TODO 里。
>
> 文档索引：`docs/README.md`。

## 0) 已决定的约束（写死，别再争论）

- **Resume 语义（Codex 风格）**：一个 session 必须包含完整上下文：对话、tool/进程步骤、中间态产物、审批与结果；`resume` 时用这些历史拼合上下文并继续对话。**不承诺“原进程继续跑”**，但必须承诺“历史不丢、可继续推进”。
- **Artifact 定义**：repo/workspace 里的代码变更属于“repo 更新”，不叫 artifact。artifact = **给用户看的文档** + **不进 repo 的临时产物**（由 agent 判断生成/归档），参考 `~/.gemini/antigravity/brain/*/*.md + *.metadata.json` 的形态。
- **进程 attach**：只允许 `inspect/tail/follow`（只读查看）或 `kill`，**禁止交互输入**（不做 stdin/PTY 交互）。
- **模型路由优先级（已定）**：`subagent 强制 > project override > keyword rule > role default > global default`，且历史记录必须能看到“本次用了什么模型、为什么”。
- **跨 session 记忆（可选）**：默认不启用；未来只记录“用户特征/偏好/擅长领域/常用库/代码风格”等 profile，不默认记代码内容。

## 0.1) 单 agent CLI（无编排，仅 system prompt）可用性

结论：**v0.2.0 已可跑通**“一个 thread → 一个 turn → tool side-effects → JSONL 落盘/回放”的最小闭环（无需 role 路由、无需 workflow/orchestrator）。

最小前置（可复制）：

```bash
export OPENAI_API_KEY=...
cargo run -p pm -- exec "write a file named hello.txt"
```

交互式（REPL）：

```bash
export OPENAI_API_KEY=...
cargo run -p pm
```

system prompt（`instructions`）组成（v0.2.0 现状）：

- 内置默认 instructions：`crates/app-server/src/agent/core.rs` 的 `DEFAULT_INSTRUCTIONS`
- 可选 user instructions：`CODE_PM_USER_INSTRUCTIONS_FILE` 或 `~/.codepm_data/AGENTS.md`
- 可选 project instructions：thread cwd 下 `AGENTS.md`
- 可选 skills：按 `$skill-name` 触发注入

## B. 核心语义（必做）

- `Thread/Turn/Item` 强类型事件模型
  - 已定：**append-only 落盘事件**是权威来源；流式通知只是优化，断线/lag 可用 `resume + 从 offset/seq 重放` 补齐。
  - 已定：每个 `Item`/`ProcessEvent` 必须有**单调递增序号**（同一 thread 内）用于重放/去重/拼合历史。
  - 已定：订阅可靠性（最小语义）：
    - 客户端订阅使用 `since_seq`（从 `since_seq + 1` 重放/继续），并持久化自己的 `last_seq`（掉线后能续）。
    - 服务端按 `seq` 有序推送；允许重复（at-least-once），客户端按 `seq` 去重即可。
    - 不用 ack 作为正确性前提：**不丢的保证来自落盘 log**，订阅只是投递优化；消费端 lag 了就重连补齐。
  - [x] Item 覆盖（详见 `docs/thread_event_model.md`）
    - [x] message：`AssistantMessage` + `item/delta`（文本流）
    - [ ] reasoning：v0.2.0 不默认落盘（后续用 summary artifact，走脱敏；TODO 草案见 `docs/budgets.md`）
    - [x] tool：`ToolStarted/ToolCompleted`
    - [x] file edit：`file/write|edit|patch|delete|fs/mkdir`（只记元信息，不记内容）
    - [x] diff：v0.2.0 先作为 artifact（`thread/diff` → user artifact `artifact_type="diff"`；CLI: `pm thread diff <thread_id>`；见 `docs/artifacts.md`）
    - [x] approval：`ApprovalRequested/ApprovalDecided`
    - [x] process：`ProcessStarted/Exited/...`（stdout/stderr 为 artifacts）
  - [x] 落盘：JSONL rollout + 回放（replay）+ 最小索引（list/show）
  - [x] 派生视图：Attention/Inbox（v0.2.0：need_approval/running/failed/stuck/done/idle/paused/archived/interrupted/cancelled；见 `docs/attention.md`）
  - 长任务防失控（详见 `docs/budgets.md`）
    - [x] budget/timeout：steps/tool_calls/turn_seconds/openai_request_seconds/total_tokens → `TurnStatus::Stuck`
